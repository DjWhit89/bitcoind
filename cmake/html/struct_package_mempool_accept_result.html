◆ m_tx_results

std::map<Wtxid, MempoolAcceptResult> m_tx_results

Map from wtxid to finished MempoolAcceptResults. The client is responsible for keeping track of the transaction objects themselves. If a result is not present, it means validation was unfinished for that transaction. If there was a package-wide error (see result in m_state), m_tx_results will be empty.
