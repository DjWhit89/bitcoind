\doxysubsection{Detailed Description}
A wrapper to reserve an address from a wallet

\doxyref{Reserve\+Destination}{p.}{classwallet_1_1_reserve_destination} is used to reserve an address. It is currently only used inside of Create\+Transaction.

Instantiating a \doxyref{Reserve\+Destination}{p.}{classwallet_1_1_reserve_destination} does not reserve an address. To do so, \doxyref{Get\+Reserved\+Destination()}{p.}{classwallet_1_1_reserve_destination_a524758fb27e78dd234cdb604c26dad5d} needs to be called on the object. Once an address has been reserved, call \doxyref{Keep\+Destination()}{p.}{classwallet_1_1_reserve_destination_a149c1909365fa60f577ed0894a6d050f} on the \doxyref{Reserve\+Destination}{p.}{classwallet_1_1_reserve_destination} object to make sure it is not returned. Call \doxyref{Return\+Destination()}{p.}{classwallet_1_1_reserve_destination_a7fcbe7d63fc692d9b0b109f0d79e95aa} to return the address so it can be reused (for example, if the address was used in a new transaction and that transaction was not completed and needed to be aborted).

If an address is reserved and \doxyref{Keep\+Destination()}{p.}{classwallet_1_1_reserve_destination_a149c1909365fa60f577ed0894a6d050f} is not called, then the address will be returned when the \doxyref{Reserve\+Destination}{p.}{classwallet_1_1_reserve_destination} goes out of scope.
