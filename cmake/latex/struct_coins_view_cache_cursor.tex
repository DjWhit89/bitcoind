\doxysubsection{Detailed Description}
Cursor for iterating over the linked list of flagged entries in \doxyref{CCoins\+View\+Cache}{p.}{class_c_coins_view_cache}.

This is a helper struct to encapsulate the diverging logic between a non-\/erasing \doxyref{CCoins\+View\+Cache\+::\+Sync}{p.}{class_c_coins_view_cache_a071d010e76ac762ffab420fcf90656a4} and an erasing \doxyref{CCoins\+View\+Cache\+::\+Flush}{p.}{class_c_coins_view_cache_a166e3b1951caaaf582e7b67c5be762ce}. This allows the receiver of \doxyref{CCoins\+View\+::\+Batch\+Write}{p.}{class_c_coins_view_a433922bb29053e30cb00224aa60351e7} to iterate through the flagged entries without knowing the caller\textquotesingle{}s intent.

However, the receiver can still call \doxyref{Coins\+View\+Cache\+Cursor\+::\+Will\+Erase}{p.}{struct_coins_view_cache_cursor_aa0202b2599a0d8d029f5a8f277d7dd31} to see if the caller will erase the entry after Batch\+Write returns. If so, the receiver can perform optimizations such as moving the coin out of the CCoins\+Cach\+Entry instead of copying it.

\doxysubsubsection{CoinsViewCacheCursor()}
{\footnotesize\ttfamily \label{struct_coins_view_cache_cursor_a567c0cfee2acda924e520264c573f44b} 
\textbf{ Coins\+View\+Cache\+Cursor} (\begin{DoxyParamCaption}\item[{\textbf{ Coins\+Cache\+Pair} \&sentinel}]{LIFETIMEBOUND}{, }\item[{\textbf{ CCoins\+Map} \&map}]{LIFETIMEBOUND}{, }\item[{bool}]{will\+\_\+erase}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}, {\ttfamily [noexcept]}}

If will\+\_\+erase is not set, iterating through the cursor will erase spent coins from the map, and other coins will be unflagged (removing them from the linked list). If will\+\_\+erase is set, the underlying map and linked list will not be modified, as the caller is expected to wipe the entire map anyway. This is an optimization compared to erasing all entries as the cursor iterates them when will\+\_\+erase is set. Calling CCoins\+Map\+::clear() afterwards is faster because a \doxyref{Coins\+Cache\+Pair}{p.}{coins_8h_a4f73235f01457f00d91f656dffdb6235} cannot be coerced back into a CCoins\+Map\+::iterator to be erased, and must therefore be looked up again by key in the \doxyref{CCoins\+Map}{p.}{coins_8h_abb6c00f4f94a3084a430d1848fed7697} before being erased.
