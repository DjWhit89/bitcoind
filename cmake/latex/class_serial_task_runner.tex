\doxysubsection{Detailed Description}
Class used by \doxyref{CScheduler}{p.}{class_c_scheduler} clients which may schedule multiple jobs which are required to be run serially. Jobs may not be run on the same thread, but no two jobs will be executed at the same time and memory will be release-\/acquire consistent (the scheduler will internally do an acquire before invoking a callback as well as a release at the end). In practice this means that a callback \doxyref{B()}{p.}{util__tests_8cpp_a111da81ae5883147168bbb8366377b10} will be able to observe all of the effects of callback A() which executed before it.
