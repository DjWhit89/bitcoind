\label{_2test_2result_tests_8cpp-example}The \doxyref{util\+::\+Result}{p.}{classutil_1_1_result} class provides a standard way for functions to return either error messages or result values.

It is intended for high-\/level functions that need to report error strings to end users. Lower-\/level functions that don\textquotesingle{}t need this error-\/reporting and that only need error-\/handling should avoid \doxyref{util\+::\+Result}{p.}{classutil_1_1_result} and instead use \doxyref{util\+::\+Expected}{p.}{classutil_1_1_expected}, std\+::optional, std\+::variant, or custom structs and enum types to return function results.

Usage examples can be found in, but in general code returning {\ttfamily \doxyref{util\+::\+Result}{p.}{classutil_1_1_result}$<$\doxyref{T}{p.}{hash__tests_8cpp_ad4971a985148c784a362f962d42aa012}$>$} values is very similar to code returning {\ttfamily std\+::optional$<$\doxyref{T}{p.}{hash__tests_8cpp_ad4971a985148c784a362f962d42aa012}$>$} values. Existing functions returning {\ttfamily std\+::optional$<$\doxyref{T}{p.}{hash__tests_8cpp_ad4971a985148c784a362f962d42aa012}$>$} can be updated to return {\ttfamily \doxyref{util\+::\+Result}{p.}{classutil_1_1_result}$<$\doxyref{T}{p.}{hash__tests_8cpp_ad4971a985148c784a362f962d42aa012}$>$} and return error strings usually just replacing {\ttfamily return std\+::nullopt;} with {\ttfamily return \doxyref{util\+::\+Error}{p.}{structutil_1_1_error}\{error\+\_\+string\};}.
