Setting n\+Sequence to this value for every input in a transaction disables n\+Lock\+Time/\+Is\+Final\+Tx(). It fails OP\+\_\+\+CHECKLOCKTIMEVERIFY/\+Check\+Lock\+Time() for any input that has it set (BIP 65). \doxysubsubsection{SEQUENCE\_LOCKTIME\_GRANULARITY}
{\footnotesize\ttfamily \label{class_c_tx_in_af3e53090c7ed2023646e1bb85c23998b} 
const int SEQUENCE\+\_\+\+LOCKTIME\+\_\+\+GRANULARITY = 9\hspace{0.3cm}{\ttfamily [static]}}

In order to use the same number of bits to encode roughly the same wall-\/clock duration, and because blocks are naturally limited to occur every 600s on average, the minimum granularity for time-\/based relative lock-\/time is fixed at 512 seconds. Converting from \doxyref{CTx\+In\+::n\+Sequence}{p.}{class_c_tx_in_a226d890a323aa88e91b64ec9e2ab280d} to seconds is performed by multiplying by 512 = 2$^\wedge$9, or equivalently shifting up by 9 bits. \index{CTxIn@{CTxIn}!SEQUENCE\_LOCKTIME\_MASK@{SEQUENCE\_LOCKTIME\_MASK}}
