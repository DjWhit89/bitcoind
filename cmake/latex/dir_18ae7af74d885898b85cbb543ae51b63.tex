\doxysubsection{Detailed Description}
\doxysubsection{src/node/}\label{dir_18ae7af74d885898b85cbb543ae51b63_autotoc_md100}
The {\texttt{{\ttfamily src/node/}}} directory contains code that needs to access node state (state in {\ttfamily \doxyref{CChain}{p.}{class_c_chain}}, {\ttfamily \doxyref{CBlock\+Index}{p.}{class_c_block_index}}, {\ttfamily \doxyref{CCoins\+View}{p.}{class_c_coins_view}}, {\ttfamily \doxyref{CTx\+Mem\+Pool}{p.}{class_c_tx_mem_pool}}, and similar classes).

Code in {\texttt{{\ttfamily src/node/}}} is meant to be segregated from code in {\texttt{{\ttfamily src/wallet/}}} and {\texttt{{\ttfamily src/qt/}}}, to ensure wallet and GUI code changes don\textquotesingle{}t interfere with node operation, to allow wallet and GUI code to run in separate processes, and to perhaps eventually allow wallet and GUI code to be maintained in separate source repositories.

As a rule of thumb, code in one of the {\texttt{{\ttfamily src/node/}}}, {\texttt{{\ttfamily src/wallet/}}}, or {\texttt{{\ttfamily src/qt/}}} directories should avoid calling code in the other directories directly, and only invoke it indirectly through the more limited {\texttt{{\ttfamily src/interfaces/}}} classes.

This directory is at the moment sparsely populated. Eventually more substantial files like {\texttt{{\ttfamily src/validation.\+cpp}}} and {\texttt{{\ttfamily src/txmempool.\+cpp}}} might be moved there.
